# Sessions

This folder is the project's history. Every meaningful unit of work — an architect brief from Cowork, an implementation report from Claude Code — is one Markdown file here.

For the format, naming convention, status lifecycle, and full rules, see [`docs/agents/SESSIONS.md`](../docs/agents/SESSIONS.md).

## Quick reference

- **Naming:** `YYYY-MM-DD-NNN-{arch|impl}-{slug}.md` — e.g. `2026-04-28-002-arch-phase-1-handoff.md`
- **`NNN`** is monotonically increasing across the whole project (not reset daily)
- **Templates** live in `_templates/` — copy when starting a new session
- **Drafts** go in `_drafts/` (gitignored)
- **Archive:** when a phase wraps, move completed sessions to `archive/YYYY-MM/`

- **Redirect-Mechanismus: HTTP 307 vs. meta-refresh nach `loading.tsx`-Add (PR #11 / Stufe 2c.1).** Mit `src/app/timeline/loading.tsx` (instant Skeleton beim Hub→Timeline-Click) hat sich das Verhalten von `redirect()`-Calls in `page.tsx` geändert: Stream startet vor dem Redirect → Next emittet `<meta http-equiv="refresh" content="1;url=…">` plus RSC-Redirect-Direktive, statt sauberen HTTP 307. Browser folgen korrekt; JS-User kriegen den Redirect via Next-Router instant; **Direkthits auf Redirect-URLs (geteilte Links, Legacy `?era=M30`) bekommen 1 Sekunde Skeleton-Blink vor Meta-Refresh**. Sauberere Alternative: Redirect-Logik in `middleware.ts` ziehen (läuft vor Page-Render, gibt 307 sofort zurück). Trade-off: ~80 Zeilen `middleware.ts` mit 1–2 DB-Queries vor jedem `/timeline`-Hit. Mit Philipp am 2026-05-02 bestätigt: bleibt vorerst hier liegen — Internal-Nav ist unaffected, Direkthits sind selten genug. Eigener Brief wenn der Schmerz konkret wird.
- **Übrige `series.totalPlanned`-Werte (jenseits der 5 in Brief 034 hardgefixten).** Brief 034 (Phase 3a) korrigiert `gaunts_ghosts`, `ciaphas_cain`, `hh_more` (Primarchs), `space_wolves_sw`, `siege_of_terra` per Hand in `series.json`. Erwartung: alle weiteren falschen `totalPlanned`-Werte werden vom Pipeline-Crawler in Phase 3b (Open Library / Lexicanum-Series-Pages) automatisch gefunden und korrigiert. Falls die Crawler das Series-Total **nicht** mit-extrahieren, wandert das in einen eigenen Sammelposten-Brief nach Phase 3b.
- **Phase-3-Scale-Tuning für Timeline-Loader** (Report 030 § „For next session"). Zwei zusammengehörige Optimierungen, die heute keinen Effekt haben (26 Bücher, Planner sane), aber bei Phase-3-Skala (200+ Bücher pro Era) auflaufen: (1) `book_details.primary_era_id` ist nur FK, nicht indexiert — die EXISTS-anchored `loadEraBooks`-Queries würden bei dichten Eras von `index("book_details_primary_era_idx")` profitieren. Aufgreif-Punkt: Phase 3d (Apply-Step, dort kommt die erste Phase-3-Migration: UNIQUE INDEX `external_links` + `junctionsLocked`-Flag), oder spätestens wenn die DB nach Phase 3e tatsächlich auf 800 Bücher anwächst.
