## Decisions I made

- **Flat contract from `ask-questions.json`.** The canonical answer object is `AskAnswers = { experience?, faction_love?, tone?, length?, era_pref? }`. There is no path ID, no nested step model, and no six-path dependency in the Batch contract. Current option IDs are `experience: new|some|deep`, `faction_love: imperium|heretic|loyalist_sm|inquisition|guard|xenos`, `tone: grimdark|heroic|political|military|mythic`, `length: standalone|trilogy|epic`, and `era_pref: heresy|long_war|indomitus|any_era`.
- **`askPaths.ts` stays Product/P7 territory.** I did not touch `src/lib/askPaths.ts`. It still carries the six-path design-funnel copy and placeholder sample books for the current Product UI. Product 121-P3 should decide whether to retire it, map it onto the flat contract, or keep it as a visual staging layer.
- **121-P3 `recommend()` contract.** Product should call `recommend(answers, options?)` only from server code, because it imports the Drizzle DB client. Omit `cacheBooks` in UI code; it exists for audit performance. Default `limit` is 5. `curation: null` disables the overlay for explicit comparison; default behavior applies `scripts/seed-data/ask-curation.json`. The result is `{ answers, profile, recommendations }`, where each recommendation includes `slug`, `title`, `authors`, `score`, `matchedTags`, structured `reasons`, and optional book metadata (`primaryEraId`, `format`, `releaseYear`, `rating`, `synopsis`, `coverUrl`) plus optional `curation` effects.
- **Audit path.** The full audit writes to `ingest/ask/audit-ask-combinations.md` and `ingest/ask/audit-ask-combinations.json`. It is review output only, not frontend data. The current run covers 1080 combinations at Top-10.
- **Overlay format.** `scripts/seed-data/ask-curation.json` is `version: 1` plus `rules[]`. Each rule has `id`, partial-profile `when`, `action`, optional `points`, optional `note`, and either `book` or `tag` depending on action. Supported actions: `ban` removes a matching book, `pin` structurally sorts a book ahead for that partial profile and adds points, and `boost` adds points to a matching book or to a matched tag-axis. Tag boosts only reinforce already matched base reasons.
- **Example rules are included.** The overlay currently has five seed rules: ban `the-stacks` for the newcomer/Inquisition/political slice, pin `xenos` there, boost `eisenhorn-omnibus` for Inquisition trilogy, pin `the-infinite-and-the-divine` for xenos standalone, and boost the Guard/military axis.
